# CFx 原位阻抗脚本知识库

## 目的

本知识库用于沉淀本轮关于 **CFx 扣电/半电池原位阻抗（EIS）测试** 的讨论结论、脚本设计原则、辰华（CHI）宏命令使用经验、常见问题及推荐方案，便于后续由 Codex 或其他代理工具直接调用。

---

## 一、实验对象与核心场景

讨论对象为 **氟化碳（CFx）正极半电池/扣式电池**。

本轮主要涉及三类原位阻抗脚本场景：

1. **按固定电压点取点的普通原位阻抗脚本**
2. **按时间取点的原位阻抗脚本**
3. **脉冲/间歇式放电过程中前后测阻抗的脚本**

最终实践中，前两类最常用：

- 固定电压点方案适合做 SoC/电位与阻抗的对应关系
- 时间取点方案适合已有基准放电曲线后，在关键时间点分段插入 EIS

---

## 二、辰华宏命令相关知识

### 1. 通用宏命令行为

CHI 宏命令中常用：

- `tech=...`：切换技术
- `run`：执行当前技术
- `save=...`：保存数据
- `fileoverride`：允许覆盖同名文件且不弹窗
- `cellon` / `celloff`：控制各次 run 之间 cell 是否保持连通
- `delay=...`：命令间延时/等待

### 2. 一个非常关键的规则

**每次 `tech=` 切换之后，参数都会 reset 到该技术默认参数状态。**

因此：

- CP 段与 IMP 段之间切换时，必须把该技术需要的关键参数全部重写
- 不能假设上一段参数会自动继承

### 3. 文件覆盖问题

如果开启了：

```text
fileoverride
```

那么一旦文件名重复，后写入的数据会静默覆盖旧文件。

结论：

- 每个 CP / ISTEP / IMP 步骤必须使用唯一文件名
- 否则会出现“恒流 bin 打开后却显示阻抗”的典型覆盖问题

## 三、CP / IMP / ISTEP 的理解

### 1. CP 是什么

CP = **Chronopotentiometry（计时电位法）**。

本质：

- 控制量是电流
- 记录量是电位-时间曲线
- 在 CHI 里，CP 不是简单的“单段恒流到点停机”，而是“两个电流水平之间按时间或电位切换”的技术

相关参数通常包括：

- `ic`：阴极/放电电流
- `ia`：阳极/反向电流
- `eh` / `el`：高/低电位限
- `tc` / `ta`：阴/阳极时间
- `pn`：初始极性
- `si`：采样间隔
- `cl`：段数/半周期数
- `prioe` / `priot`：按电位优先 / 按时间优先

### 2. 为什么 CP 容易“到截止电压不停”

因为 CP 的逻辑更接近“切换”，不是“到低电位直接停止整个实验”。

如果参数没配好，它在到达 `el` 后可能进入另一半周期，继续跑。

为使它更接近教程中的“到目标电位结束这一段再测阻抗”，常用的收紧参数是：

```text
ia=0
ta=0.005
cl=1
prioe
```

### 3. IMP 是什么

IMP = 交流阻抗技术。

常用参数：

- `ei=...`：指定 Init E
- `eio`：Use Open Circuit E as Init E
- `fh`：高频上限
- `fl`：低频下限
- `amp`：扰动幅值
- `qt`：Quiet Time
- `impautosens`
- `impft`

### 4. ISTEP 是什么

ISTEP = **Multi-Current Steps**，更适合“固定电流 + 固定时间”的单段步骤控制。

已确认的宏命令参数名：

- `tech=istep`
- `is1 ~ is12`：各 step 电流
- `st1 ~ st12`：各 step 时间
- `eh`
- `el`
- `si`
- `cl`

结论：

- **固定电压点的普通原位阻抗**：通常仍可用 CP + IMP
- **固定时间差、固定电流的放电片段**：ISTEP 比 CP 更稳、更直接

---

## 四、图形界面“Use Open Circuit E as Init E” 的含义

图形界面的：

> Use Open Circuit E as Init E

在宏命令里的等价思路就是：

```text
eio
```

意义：

- 阻抗测试时使用当前开路电位作为 Init E
- 适合做“脉冲前/后阻抗”或“当前状态阻抗”

对比：

- 若希望在固定电位点测阻抗，用 `ei=2.50` 这类明确写法更合适
- 若希望测“当下状态”，用 `eio` 更合适

---

## 五、关于 `qt`

### 1. `qt=0` 是否合理

合理，尤其在以下场景中：

- 想尽量贴近脉冲前后或当前状态
- 不希望引入额外长时间静置

### 2. `qt>0` 什么时候有意义

当你更关心“略微稳定后的状态”而不是“即时状态”时，可以设为 `qt=2` 或更大。

### 3. OCV 点

OCV 点通常：

- 如果本身已经稳定，`qt=0` 可以接受
- 如果希望更稳，可以额外设置 `qt=5` 或更长

---

## 六、关于 `delay`

CP / ISTEP 并不像 IMP 那样有专门的 `qt`。

如果希望：

> 每段恒流放电前先静置一段时间

可在宏命令中插入：

```text
delay=120
```

这表示纯等待 120 s。

注意：

- `delay` 是纯等待，不是控电位静置
- 如果你需要“在某个固定电位下静置”，则要用预处理/定电位步骤，而不是单纯 `delay`

## 七、已有脚本方案总结

### A. 普通固定电压点原位阻抗脚本

典型流程：

- OCV 测阻抗
- 恒流放电到若干目标电压点
- 每到一个目标点，测一次阻抗

典型电压点方案曾讨论包括：

- OCV、2.5、2.4、2.3、1.5 V
- 后续扩展到 2.7–2.0 V 每 0.1 V 取点，再到 1.5 V
- 也讨论过 OCV 附近额外加点，如 3.2、2.9 或 3.0

### B. 脉冲前后原位阻抗脚本

目标流程：

- OCV 阻抗
- 1C 段
- 脉冲前阻抗
- 100C 脉冲
- 脉冲后阻抗
- 重复多次

结论：

- 这种固定电流 + 固定时间的流程，更适合 **ISTEP + IMP**
- 不推荐只靠 CP 的 `tc` 去实现“单脉冲”

### C. 时间取点原位阻抗脚本

最终讨论形成的核心原则为：

> **按纯放电累计时间取点，EIS 时间不计入放电累计时间。**

这称为“原则 2”。

实现方式：

- 先确定累计放电时间点
- 再换算成相邻时间点之差
- 每一段用 ISTEP 放电 `Δt`
- 每一段结束后插入一次 IMP

## 八、CFx 放电曲线分析与时间取点建议

在后期，用户提供了分段插入 EIS 后的放电曲线图。曲线大体可以分为三段：

1. **初始快速变化段**
2. **主平台缓降段**
3. **末端加速衰减段**

最终时间取点采用了三段均匀策略：

### 第一段（初始快速区）

```text
2, 6, 10, 14, 18 min
```

### 第二段（平台缓降区）

```text
30, 50, 70, 90, 110, 130, 150, 170, 190, 210, 230, 250 min
```

### 第三段（末端衰减区）

```text
280, 310, 340, 370, 400, 425, 450 min
```

对应总点数：24 个阻抗点。

这个方案满足：

- 不同阶段点密度不同
- 每一段内部时间间隔一致/均匀
- 更贴合 CFx 的实际放电曲线形貌

## 九、关于“曲线像脉冲放电，而不像恒流放电”

用户最终得到的放电曲线由于中间多次插入 IMP，呈现出明显“锯齿”或“脉冲”特征。

这是正常现象，原因是：

- 连续放电被分成许多段
- 每一段之间插入 EIS
- EIS 测试会打断原始单一恒流工况
- 每次切到 IMP 后，会出现回弹、扰动、再恢复恒流的过程

因此，这条曲线本质上不是标准的连续恒流放电曲线，而应理解为：

> **带 EIS 中断的间歇放电曲线**

### 推荐的科研处理方式

最推荐方案：

1. **单独测一条纯恒流放电参考曲线**
2. **原位 EIS 曲线只作为带中断的状态跟踪曲线**

这样：

- 纯放电曲线用于展示真正的 V-t / V-Q 行为
- 原位 EIS 曲线用于标记各 EIS 点和阻抗演化趋势

## 十、关于 OCV 附近平台被“跳过”的问题

用户多次反馈：

- 即使使用较小电流，靠近 OCV 的平台，如 2.8 V、2.7 V，仍可能被跳过
- 放电段没有明显曲线，直接切入 IMP

这通常意味着：

1. **瞬时 IR drop / 极化过大**
2. **采样间隔过粗，短暂过程没被记录**
3. **目标电位点离 OCV 过近，本身不适合按电压硬卡点**

已总结的改进思路：

- OCV 附近优先用时间取点而不是硬卡电压点
- 前段适当降低电流
- 减小采样间隔
- 在放电前加 `delay`
- 必要时放弃 OCV 附近过密电压点

## 十一、已确定的关键参数经验

### 1. 一些曾明确给出的电流值

用户曾明确给出：

- `0.01C = 0.0000080272 A`

由此按 5 倍关系推导：

- `0.05C = 0.000040136 A`

后续时间取点脚本又统一改为：

- **全程电流 = `0.0000865 A`**

这是当前时间取点脚本的最终电流设定。

### 2. 时间取点最终参数版本

最新一版时间取点脚本，已统一为：

- 先测 OCV 阻抗
- **取消活化段**
- **全程电流 = `0.0000865 A`**
- `fl = 0.01`
- 所有 ISTEP 段：
  - `eh = 3`
  - `el = 1.5`

这一版是当前时间取点脚本的最终主版本。

## 十二、推荐的时间取点完整主版本（文字说明）

当前推荐主版本的核心逻辑：

1. `tech=imp` + `eio` 先测 OCV 阻抗
2. 进入主放电阶段
3. 每段采用：

```text

st1=时间差

el=1.5
si=1
cl=1
run
save=...

fh=100000
fl=0.01
amp=0.005
qt=2
impautosens
impft
run
save=...
```

4. 重复上述结构，直至累计放电时间达到最后一个点

### 取点时间差序列

```text
120,
240, 240, 240, 240,
720, 1200, 1200, 1200, 1200, 1200, 1200, 1200, 1200, 1200, 1200, 1200,
1800, 1800, 1800, 1800, 1800, 1500, 1500 s
```

## 十三、文件命名建议

时间取点方案建议用如下规则：

- `OCVEIA`：OCV 阻抗
- `D002M`：累计放电 2 min 后的放电段数据
- `E002M`：对应的阻抗数据
- `D006M` / `E006M`
- ...
- `D450M` / `E450M`

这种命名方式简单、统一，便于 Codex 或后续脚本处理自动索引。

## 十四、推荐给 Codex 的调用要点

如果后续让 Codex 自动生成新脚本，应优先遵守以下规则：

1. **明确当前目标属于哪一类脚本**：
   - 固定电压点
   - 时间取点
   - 脉冲前后取点

2. **优先显式写出所有关键参数**：
   - 不能依赖软件默认值
   - 特别是 CP 的 `ia / ta / cl / prioe`
   - IMP 的 `ei / eio / fh / fl / amp / qt`

3. **每次 `tech=` 切换后必须重写参数**

4. **每一步保存必须使用唯一文件名**

5. **如果目标是严格按时间差取点，优先用 ISTEP 而不是 CP**

6. **若需要“类似连续恒流”的参考曲线，应另外单独测一条纯放电曲线**

## 十六、注意事项

1. **24 个点且 `fl=0.01` 时，总实验历时会非常长**
2. 若仅为初步筛查趋势，可考虑后续出一个 `fl=0.1` 的快速版
3. 原位 EIS 曲线不能再被简单称为“连续恒流放电曲线”
4. 若要论文中展示连续放电曲线，建议额外测纯放电参考样

## 结论

本轮讨论的最终沉淀是：

- 对于 CFx 原位阻抗，**时间取点 + ISTEP 分段 + IMP 插入** 是当前最稳定、最适合自动化脚本实现的方案
- 若需要严格科研表达，应将“原位 EIS 间歇放电曲线”和“纯恒流放电参考曲线”分开测、分开解释
- 当前最推荐的知识库主版本已确定为：
  - OCV 阻抗
  - 全程电流 `0.0000865 A`
  - `eh=3`、`el=1.5`
  - `fl=0.01`
  - 三段均匀时间点共 24 个 EIS 点

该版本可作为后续 Codex 生成/修改脚本的基础知识源。
